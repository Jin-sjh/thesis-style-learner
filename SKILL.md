# 毕业论文风格蒸馏与写作技能

## 核心原则

1. **只学风格，不抄内容**：从示例论文中提取写作模式，绝不复制原文句子或段落。
2. **完整风格画像**：分析 6 个维度（句式、段落逻辑、语气、术语、格式、章节范式）。
3. **可操作输出**：生成的内容可直接用于学术写作，符合中文/英文学术规范。
4. **大文件优先策略**：论文 PDF 通常体积较大，采用分层读取策略，避免一次性全量加载。

---

## 工作流程

### 阶段一：接收与预处理示例论文

用户会提供一篇或多篇参考论文。按以下步骤处理：

#### 1A. 文件类型判断

- **PDF 文件**：参考 [PDF 处理策略](#pdf-处理策略) 章节
- **纯文本/复制粘贴**：直接进入阶段二分析
- **多篇论文**：分别蒸馏后合并风格特征，注明共性与差异

#### 1B. PDF 处理策略

**⚠️ 重要：PDF 是二进制文件，不能直接读取，必须先用工具提取文本。**

在处理任何论文 PDF 前，**必须先读取** `/mnt/skills/public/pdf-reading/SKILL.md`，获取完整的 PDF 提取方法，然后按以下专项流程执行：

**步骤 1：诊断 PDF 类型**
```bash
# 检查页数、元数据
pdfinfo /mnt/user-data/uploads/<filename>.pdf

# 检查文本是否可提取（关键：区分"文字版"和"扫描版"）
pdftotext -f 1 -l 2 /mnt/user-data/uploads/<filename>.pdf - | head -30
```

- 若有输出 → 文字版 PDF，走路径 A
- 若输出为空或乱码 → 扫描版 PDF，走路径 B

**路径 A：文字版 PDF（最常见）**
```bash
# 提取全文到临时文件
pdftotext -layout /mnt/user-data/uploads/<filename>.pdf /tmp/thesis_full.txt

# 查看文件大小，决定采样策略
wc -c /tmp/thesis_full.txt
```

然后用 Python 按章节切分：
```python
with open('/tmp/thesis_full.txt', 'r', errors='replace') as f:
    text = f.read()

# 按字数决定策略
char_count = len(text)
if char_count < 60000:
    # 全量读取
    content = text
else:
    # 关键词定位章节边界，提取高价值区域
    import re
    sections = {}
    # 常见章节标题模式（中英文兼容）
    patterns = [
        r'(摘\s*要|Abstract)',
        r'(第?一?章?\s*引\s*言|1[\.\s]+Introduction)',
        r'(相关工作|Related Work)',
        r'(实验|Experiment)',
        r'(结论|Conclusion)',
    ]
    for pat in patterns:
        m = re.search(pat, text, re.IGNORECASE)
        if m:
            start = m.start()
            sections[pat] = text[start:start+3000]  # 每章取前3000字
    content = '\n\n'.join(sections.values())

print(content[:500])  # 验证提取结果
```

**路径 B：扫描版 PDF（图片型）**

扫描版无法直接提取文字，需要逐页视觉识别：
```bash
# 将关键页面转为图片（优先转摘要页、引言页）
pdftoppm -jpeg -r 150 -f 1 -l 5 /mnt/user-data/uploads/<filename>.pdf /tmp/thesis_page
ls /tmp/thesis_page-*.jpg
```
然后使用 `view` 工具逐页读取图片，从中提取文字内容进行风格分析。每次最多读取 5–8 页，避免 token 超限。

**路径 C：pdfplumber 备选（文字提取有乱码时）**
```bash
pip install pdfplumber --break-system-packages -q 2>/dev/null
```
```python
import pdfplumber

with pdfplumber.open('/mnt/user-data/uploads/<filename>.pdf') as pdf:
    total = len(pdf.pages)
    print(f"共 {total} 页")
    # 采样：首页、第5页、中间页、倒数第5页
    sample_pages = [0, 4, total//2, total-5, total-1]
    for i in sample_pages:
        if 0 <= i < total:
            t = pdf.pages[i].extract_text()
            if t:
                print(f"=== 第{i+1}页 ===\n{t[:800]}\n")
```

**大文件采样优先级**（按写作风格密度从高到低）：
1. 摘要 / Abstract（最高密度，必读）
2. 引言 / Introduction（范式最典型，必读）
3. 相关工作 / Related Work（术语与引用风格）
4. 结论 / Conclusion（总结范式）
5. 实验 / Experiments（数据描述语言，按需读取）

### 阶段二：风格蒸馏分析

从提取的文本中，系统性地分析以下 6 个维度。将分析结果整理为**风格档案（Style Profile）**。

#### 维度 1：句式结构

分析以下特征：
- 平均句子长度（短句/长句倾向）
- 主动句 vs 被动句的比例
- 是否大量使用"首先…其次…最后…"等连接词
- 括号补充说明的频率
- 是否倾向于使用分号或破折号连接复合句

**输出格式**：
> 句式风格：该论文倾向于使用 [长/中/短] 句，[主动/被动] 语态占主导，约 [X]% 的段落使用显式序列连接词。

#### 维度 2：段落展开逻辑

分析每段的内部结构：
- 总-分-总 / 总-分 / 分-总 哪种最常见
- 是否每段有明确的主题句
- 论据呈现方式（先理论后实验、先现象后解释）
- 段落长度均值（字数）

#### 维度 3：学术语气

- 客观陈述型（"实验结果表明…"）vs 谦逊型（"本文尝试…"）
- 是否使用第一人称（"我们"/"本文"）
- 强调方式（"值得注意的是"、"特别地"等习惯短语）
- 对比竞争方法时的措辞风格（直接批评 vs 客观比较）

#### 维度 4：专业术语使用习惯

- 英文缩写首次出现时是否附中文/全称
- 专有名词是否保持英文原文
- 自定义术语的定义方式
- 引用文献时的习惯格式（上标数字、作者年份等）

#### 维度 5：章节格式

- 标题层级命名规律（第一章/1./1.1/一、）
- 图表标题的格式（"图 X.X 标题"/"Figure X"）
- 公式编号格式
- 是否有"本章小结"/"本节总结"习惯

#### 维度 6：各章节写作范式

提取每个核心章节的**开头-中间-结尾**模板：

**摘要范式**：背景句 → 问题定义 → 方法描述 → 实验结论 → 贡献声明
**引言范式**：研究背景 → 现有方法缺陷 → 本文方法 → 贡献列表 → 章节导航
**相关工作范式**：综述组织方式 → 与本文的对比方式
**实验范式**：数据集描述 → 基线方法 → 评估指标 → 结果分析
**结论范式**：工作总结 → 局限性说明 → 未来工作

### 阶段三：输出风格档案

完成分析后，以结构化格式呈现风格档案：

```
═══════════════════════════════════════
📄 论文风格档案（Style Profile）
论文标题：[提取或推断]
研究领域：[提取]
语言：中文/英文
═══════════════════════════════════════

【句式风格】
...

【段落逻辑】
...

【学术语气】
...

【术语习惯】
...

【格式规范】
...

【章节范式】
摘要：...
引言：...
（以下各章）
...

【典型表达模式】（从原文提取的句型框架，非原文内容）
- 方法介绍句型："本文提出了一种基于...的方法，通过...实现了..."
- 实验对比句型："与基线方法相比，本文方法在...指标上提升了..."
- 问题引出句型："然而，现有方法在...方面存在..."
（注：以上为结构框架，不含原文内容）
═══════════════════════════════════════
```

### 阶段四：按风格生成新内容

用户提供：
1. 研究方向/主题
2. 需要生成的章节名称
3. 关键内容要点（可选）

按以下规范生成：

**生成规则**：
- 严格按照风格档案中的句式、段落结构、语气进行写作
- 使用相同的章节范式框架，填入用户的研究内容
- 专业术语使用习惯与原论文一致
- 输出前注明："以下内容依据风格档案生成，所有学术观点和数据由用户提供，本技能仅负责语言风格。"

**质量检查**（生成后自检）：
- [ ] 无句子与原文相似度过高（逐句验证）
- [ ] 段落逻辑符合分析出的展开模式
- [ ] 学术语气一致
- [ ] 格式符合章节规范

## 注意事项与学术伦理

1. **风格 ≠ 内容**：本技能学习的是语言表达模式，用户必须自行提供真实的研究内容、数据和结论。
2. **不生成虚假实验数据**：如用户未提供实验数据，生成文本中用 `[数据待填写]` 占位。
3. **不替代学术思考**：本技能是写作辅助工具，核心研究工作需用户独立完成。
4. **引用规范**：生成的参考文献部分只提供格式模板，具体文献由用户核实填写。

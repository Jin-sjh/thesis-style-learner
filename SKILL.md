# 毕业论文风格蒸馏与内容生成助手

本技能从导师推荐的往届优秀毕业论文中深度学习写作风格，并生成风格一致的新内容。

## 核心能力

1. **多论文风格融合** - 从多篇论文中提取共同风格特征
2. **导师偏好识别** - 识别导师对表达的偏好习惯
3. **深度风格建模** - 句式、连接词、段落、论证、术语全面建模
4. **风格一致性生成** - 生成与学习风格高度一致的内容

## 工作流程

1. **接收论文** - 解析用户提供的论文（支持多篇）
2. **风格蒸馏** - 从多篇论文中提取共同风格
3. **偏好识别** - 识别导师的表达偏好
4. **风格建模** - 构建完整的风格模型
5. **质量检验** - 验证生成内容的风格一致性
6. **内容生成** - 生成符合风格的论文内容

---

## 第一步：接收论文

### 支持的输入格式

- 纯文本 (.txt)
- PDF 文件 (.pdf)  
- Word 文档 (.docx)

### 多论文处理

支持用户提供多篇论文（2-5篇为宜）：

- 所有论文应来自同一导师指导
- 包含不同研究方向的样本更佳
- 标记每篇论文的来源（学生姓名、毕业年份）

### 提取论文关键信息

从每篇论文中提取：

1. 摘要（中英文）
2. 绪论/引言
3. 相关工作/文献综述
4. 方法/模型描述
5. 实验章节
6. 结论与展望

### 批量论文处理策略

当用户提供**大量论文（>5篇）**时，采用**压缩摘要法**：

1. **并行分析** - 同时分析所有论文，提取风格特征
2. **压缩存储** - 只保留风格相关数据，丢弃具体内容
3. **增量更新** - 处理过程中实时显示进度

### 处理流程

1. **输入论文列表** → 用户提供多篇论文文件
2. **批量风格分析** → 并行提取每篇论文的风格特征
3. **生成压缩摘要** → 存储风格数据到 `style_extracts/` 目录
4. **风格融合** → 合并多篇论文的共同特征
5. **生成风格模型** → 最终的导师偏好模型

### 内存优化

- 单篇论文处理后立即释放原始文本
- 压缩摘要大小约为原文的 **1-3%**
- 支持处理 **20+ 篇论文**而不超出内存

## 第二步：风格蒸馏（核心）

### 多论文共同风格提取

当用户提供多篇论文时，需要识别**共同的风格特征**，即导师的偏好：

#### 2.1 句式长度分布

- 统计每篇论文的句子长度分布
- 计算多篇论文的**共同分布模式**
- 识别主导句长区间（如 30-50 字的中长句）
- 标记长短句交替使用的节奏

**提取项：**
- 最短句长度 / 最长句长度 / 平均句长
- 句长分布直方图
- 长短句交替频率

#### 2.2 连接词使用模式

学术写作中的连接词反映论证逻辑。

**分析以下连接词类别：**

| 类别 | 示例词汇 | 功能 |
|------|----------|------|
| 因果 | 因此、所以、由此可见、因而 | 揭示因果关系 |
| 转折 | 然而、但是、尽管如此、但 | 引入对比或反驳 |
| 递进 | 更进一步、此外、此外、并且 | 推进论证 |
| 让步 | 虽然、即使、即便 | 承认对立观点 |
| 总结 | 综上所述、总之、总之 | 归纳结论 |
| 举例 | 例如、以...为例、诸如 | 提供例证 |
| 引出 | 值得注意的是、需要指出 | 引入重点 |

**提取项：**
- 每类连接词的使用频率
- 多篇论文共同偏好的连接词
- 连接词在段落中的位置（段首、句中、过渡）

#### 2.3 段落开头方式

段落开头决定了读者的阅读体验。

**分析段落开头模式：**

1. **主题句开头** - 段首直接陈述观点
2. **背景引入** - 先描述背景再引出观点
3. **问题引导** - 以问题形式开启段落
4. **引用引入** - 以文献或数据引入
5. **对比引入** - 先描述他人观点再引出自己观点

**提取项：**
- 各开头模式的使用频率
- 不同章节的偏好开头方式
- 多篇论文共同的段落开头模式

#### 2.4 论证逻辑结构

分析段落内部的论证展开方式。

**常见论证模式：**

1. **总-分-总** - 先总述后分述最后总结
2. **总-分** - 先总述后分述
3. **分-总** - 先分述后总结
4. **因果链** - 原因→结果→原因→结果
5. **对比论证** - A观点→B观点→对比分析
6. **递进论证** - 层面1→层面2→层面3
7. **问题-解决** - 提出问题→分析问题→解决问题

**提取项：**
- 各论证模式的使用频率
- 每种模式适用的场景
- 多篇论文共同使用的论证模式

#### 2.5 引用格式分析

引用格式是学术规范的重要组成部分。

**分析项：**

1. **参考文献格式**
   - 编号顺序 vs 作者年份
   - 标点符号使用（逗号、冒号、句号）
   - 作者姓名格式
   - 期刊名称格式
   - DOI/URL 处理

2. **文中引用方式**
   - 上标格式 [1]
   - 作者年份 (Zhang, 2023)
   - 数字格式 (1) vs 上标格式 [1]
   - 引用位置（句首、句中、句尾）

3. **引用标注风格**
   - 单篇引用 [1]
   - 多篇引用 [1][2] 或 [1,2]
   - 范围引用 [1-3] 或 [1~3]
   - 全部列出 vs et al.

**常见引用格式：**

| 格式标准 | 参考文献 | 文中引用 |
|---------|----------|----------|
| GB/T 7714 | 序号+作者. 期刊, 年 | [1] |
| APA | (Author, Year) | (Author, 2023) |
| IEEE | [1] Author, "Title" | [1] |
| MLA | Author. "Title." | (Author page) |

#### 2.6 专业表述方式

学习领域中特有的表达习惯。

**分析项：**

1. **术语首次引入方式**
   - 定义式：XXX是指...
   - 引用式：根据文献[1]，XXX...
   - 描述式：通过...实现XXX

2. **模型/方法描述方式**
   - 整体到局部的描述顺序
   - 公式与文字的配合方式
   - 模块之间关系的表述

3. **实验描述方式**
   - 数据集介绍顺序
   - 对比方法的描述深度
   - 结果分析的角度

4. **图表描述方式**
   - "如图X所示"的引入方式
   - 数据描述的顺序
   - 趋势描述的词汇

## 第三步：导师偏好识别

### 从多篇论文中识别共同偏好

当有多篇论文时，找出**共同的模式**，这些就是导师的偏好：

1. **所有论文都使用的表达方式** → 导师强制偏好
2. **大部分论文使用的表达方式** → 导师倾向偏好
3. **个别论文特有的表达方式** → 学生个人风格（不作为模板）

### 偏好强度分类

| 强度 | 覆盖率 | 学习建议 |
|------|--------|----------|
| 强偏好 | ≥80%论文使用 | 必须遵循 |
| 中偏好 | 50-80%论文使用 | 尽量遵循 |
| 弱偏好 | <50%论文使用 | 参考性学习 |

### 偏好库构建

整理导师偏好的表达方式，形成偏好库：

```json
{
  "advisor_preferences": {
    "sentence_length": {
      "dominant_range": "30-50字",
      "variation": "长短交替",
      "strength": "强偏好"
    },
    "connectors": {
      "causal": ["因此", "由此可见", "因而"],
      "transitional": ["然而", "但是"],
      "progressive": ["更进一步", "此外"],
      "strength": "强偏好"
    },
    "paragraph_opening": {
      "dominant_mode": "主题句开头",
      "secondary_mode": "对比引入",
      "strength": "强偏好"
    },
    "argumentation": {
      "dominant_pattern": ["总-分", "因果链"],
      "strength": "强偏好"
    }
  }
}
```

## 第四步：风格建模

### 完整风格模型结构

将分析结果整合为完整的风格模型：

```json
{
  "style_model": {
    "source_papers": ["论文1", "论文2", "论文3"],
    "advisor": "导师姓名",
    "features": {
      "sentence": {
        "avg_length": "35字",
        "length_distribution": {"short": "20%", "medium": "60%", "long": "20%"},
        "connectors": {
          "causal": ["因此", "由此可见", "因而"],
          "transitional": ["然而", "但是"],
          "progressive": ["更进一步", "此外", "并且"],
          "conclusive": ["综上所述", "总之"]
        }
      },
      "paragraph": {
        "avg_length": "200字",
        "opening_patterns": ["��题句开头", "对比引入"],
        "topic_sentence_position": "段首"
      },
      "argumentation": {
        "patterns": ["总-分-总", "因果链", "对比论证"],
        "preferred_pattern": "总-分-总"
      },
      "terminology": {
        "intro_style": "定义式引入",
        "abbreviation_style": "首次全称加缩写"
      },
      "tone": {
        "formality": "非常正式",
        "voice": "被动语态为主",
        "certainty": "谨慎客观"
      }
    }
  }
}
```

### 风格模型的应用原则

1. **强偏好必须遵循** - 导师偏好的核心特征
2. **中偏好尽量遵循** - 体现一致性
3. **可适度创新** - 在非核心特征上可有自己的风格

## 第五步：内容生成

### 用户输入要求

用户提供：

1. **研究方向** - 论文的研究主题
2. **章节要求** - 需要生成的章节
3. **字数要求** - 目标字数
4. **核心要素** - 模型、数据集、实验设置等关键信息

### 生成过程

1. **内容规划** - 根据章节要求规划内容结构
2. **风格匹配** - 从风格模型中选择对应的模式
3. **句子生成** - 按照风格模型生成句子
4. **段落组装** - 按照段落模型组装句子
5. **风格检验** - 检查是否符合风格特征

### 风格检验清单

生成后对照风格模型检验：

- [ ] 句子长度是否在主导区间内
- [ ] 是否使用了偏好的连接词
- [ ] 段落开头是否使用了偏好的模式
- [ ] 论证逻辑是否符合偏好模式
- [ ] 术语引入方式是否符合习惯
- [ ] 语气是否符合学术规范

## 第六步：质量保证

### 风格一致性评分

生成内容后，进行风格一致性评分：

| 维度 | 分数 | 说明 |
|------|------|------|
| 句式长度 | /10 | 与模型匹配度 |
| 连接词使用 | /10 | 偏好词汇覆盖率 |
| 段落开头 | /10 | 模式匹配度 |
| 论证逻辑 | /10 | 结构匹配度 |
| 术语表述 | /10 | 习惯匹配度 |

**总分 ≥ 40/50** 视为风格一致

### 原创性检验

- 检查生成内容与原文的相似度
- 确保没有直接复制原文句子
- 如有疑问，标注供用户修改

### 输出格式

生成的内容包含：

- 章节标题
- 章节正文
- 图表占位符 [图X]、[表Y]
- 参考文献占位 [1]、[2]

用户需要填充：

- 真实文献引用
- 具体图表内容
- 实验数据
- 自己的分析见解

## 第七步：生成专属 Skill（核心功能）

将分析得到的风格模型封装为**可独立使用的 Skill**，用户可直接安装并调用。

### 7.1 生成专属 Skill 的优势

- **一键使用** - 直接调用专属风格生成内容
- **可复用** - 生成的 Skill 可多次使用
- **可分享** - 可将专属 Skill 提供给同门同学

### 7.2 Skill 结构

生成的专属 Skill 包含：

```
advisor-{导师姓名}-style/
├── SKILL.md                    # 完整的技能文件
└── references/
    └── style_model.json        # 风格模型数据
```

### 7.3 SKILL.md 模板

根据分析的风格模型，生成以下 SKILL.md：

```markdown
---
name: advisor-{导师姓}-style
description: {导师姓名}指导论文风格。当用户需要生成符合{导师姓名}指导风格的毕业论文内容时触发。使用学长的论文学习到的写作风格：{风格特征摘要}。
---

# {导师姓名}论文风格

## 风格特征

- 句式长度：{平均XX字}
- 段落开头：{主题句开头/对比引入}
- 常用连接词：{因此、此外、更进一步}
- 论证结构：{总-分-总/因果链}

## 使用方法

用户只需提供：
1. 研究方向/主题
2. 章节类型（摘要/引言/方法/实验/结论）
3. 核心内容要素

系统将自动生成符合该风格的论文内容。

## 内容生成

### 章节类型对应

| 章节 | 生成模式 |
|------|----------|
| 摘要 | 背景-问题-方法-结果-结论 |
| 引言 | 宏观背景→具体问题→研究意义 |
| 相关工作 | 分类评述，每类3-5篇文献 |
| 方法 | 系统概述→模块详解→公式算法 |
| 实验 | 数据集→对比实验→结果分析 |
| 结论 | 贡献总结→局限性→未来工作 |

### 风格要求

生成时严格遵循：
- 句子长度：{XX-XX字}
- 使用偏好连接词：{词1}、{词2}、{词3}
- 段落开头使用：{开头模式1}、{开头模式2}
- 采用{论证结构}论证

### 输出格式

生成的论文内容包含：
- 章节标题
- 正文（符合风格）
- 图表占位符 [图X]、[表X]
- 参考文献占位 [1]、[2]

用户需填充：真实实验数据、文献引用、个人见解。
```

### 7.4 生成流程

1. **收集风格特征** - 从风格模型中提取关键特征
2. **生成 SKILL.md** - 按照模板生成完整的 SKILL.md
3. **保存风格模型** - 将 style_model.json 保存到 references/
4. **输出路径** - 提示用户专属 Skill 的位置

### 7.5 输出示例

生成的专属 Skill 类似：

```
✅ 已生成专属风格 Skill！
📁 位置：C:/Users/ThinkPad/.claude/skills/advisor-zhang-style/

安装方法：将 advisor-zhang-style 文件夹复制到 Claude Skills 目录

```
